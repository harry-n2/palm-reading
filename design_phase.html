1. Database Architecture (DB設計)

3つの主要データベース（Base）のフィールド定義およびリレーション設計の詳細です。

Client Base (顧客台帳)

フィールド名 タイプ 備考
顧客ID 自動採番 Unique Key
氏名 テキスト
LINE ID テキスト 連携用キー
VIPランク 単一選択 S / A / B / 新規
生年月日 日付 運勢算出用
累計売上 計算式 Revenue Baseより集計

History Base (鑑定履歴)

フィールド名 タイプ 備考
履歴ID 自動採番
関連顧客 片方向リンク Client Baseと紐付け
鑑定日 日付
相談内容 多行テキスト
AI分析結果 多行テキスト Gemini出力先
手相写真 添付ファイル 画像URL

Revenue Base (売上管理)

フィールド名 タイプ 備考
取引NO 自動採番
購入顧客 片方向リンク Client Baseと紐付け
商品名 単一選択 鑑定60分 / 鑑定30分 etc
金額 通貨 自動入力
決済日 日付

2. Automation Logic (自動化フロー)

Lark Flowを用いた主要な自動化プロセスの定義です。

Flow B: 納品後のフォローアップ

Trigger:
納品完了 (Delivery)
↓
Wait:
20日後まで待機
↓
Action:
フォローアップメッセージ送信
(問合せフォームリンク付き)
